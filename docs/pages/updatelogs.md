## 2025/08/06

- 使用注重兼容性的方法重写了钓鱼事件的成品处理。
    - 修改了数据传递/缓存等方面，大幅优化了性能。
    - 优化了自动出售的性能，提示。现在仅支持设置为保留 (史诗、传奇、神话、世界)或以上，不再支持低品质。
    - 因钓鱼相关的互操作较多，且遍布多个系统，暂且不做完全性重构，但在未来仍有可能继续进行全流程重构。
- 对灵性鱼库略作优化，你现在可以存入自己持有的灵性鱼类了。同时，你可以在 QS 材料界面进入鱼库。
- 现在钓到的鱼王将必定是灵性。
- 现在大海垃圾库中的高价值物品将被持续保留，直到被钓走。
- 新增鱼竿附魔「轻身」、「海之眷顾」
|附魔|效果|
|:--:|--|
|轻身|降低被鱼砸晕的概率<br>晴天: 10% / lvl<br>雨天: 7.5% / lvl|
|海之眷顾|提高钓到垃圾的概率<br> +5% / lvl|
|多重垂钓|有25%概率触发多次收获<br> +1次 /lvl<br>*钓到垃圾时不生效*
|灵性|提高钓到灵性鱼类的概率<br>晴天: 25% / lvl<br>雨天: 50% / lvl|

*附魔均为在基础上乘算提升，非固定数值加成*

- 暂时移除了今日钓鱼排行功能
- 将「世界般的鱼」基础几率提升至 `0.032%`

?>
<strong>为什么一直在浪费时间重构各类东西？</strong><br>
以钓鱼为例，钓鱼主处理函数实际能耗是 几十万次/秒(μs) 级别的。<br>
但因老旧写法涉及的与其他插件查询/设置等操作，及各方面限制导致的逻辑较差 —— 不仅不雅观、难维护，往往需要消耗 500+μs，极端情况下甚至能达到 5~10ms —— 上千倍的性能差距。<br>
（尽管如此，其实不管新老，DP系统插件的能耗都是非常低的，主要性能瓶颈在MOJANG的屎）
